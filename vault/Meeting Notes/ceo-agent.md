# CEO Agent

## Overview

`.claude/agents/ceo.md` מגדיר את הסוכן הראשי של הפרויקט — **CEO Agent / מנכ"ל**. הוא subagent ייעודי של Claude Code (frontmatter `name: ceo` + description בעברית), שאליו מואצלת **כל** משימה שמגיעה מהמשתמש לפני הפעלת סוכן ייעודי.

**עקרון יסוד:** ה-CEO לא מבצע — הוא מנהל. תהליך: קבלה → הבנת מטרה עסקית → זיהוי מידע חסר ושאילת שאלות → פירוק → האצלה → בקרת איכות → אישור → תשובה סופית בפורמט קבוע.

**שבעת הסוכנים הכפופים** (כולם שטוחים תחת ה-CEO):

צוות שיווק/תוכן/קריאייטיב — קיימים כקבצים נפרדים מ-2026-05-06 (ראה [[marketing-team-split]]):
1. `content-strategist` — אסטרטג תוכן.
2. `content-writer` — כותב תוכן.
3. `copywriter` — קופירייטר.
4. `graphic-designer` — מעצב גרפי.

שאר התחומים — עדיין רק כקטגוריות ב-`ceo.md`, טרם בקבצים נפרדים:
5. הנהלת חשבונות.
6. ניהול קמפיינים ממומנים.
7. ביצוע ואופרציה.

**איסורים מוחלטים** המעוגנים בקובץ:
- אסור להחזיר תוצר שלא עבר checklist בקרת איכות.
- אסור להפעיל סוכן עם הנחות במקום לשאול.
- אסור לדלג על שלב הבנת המטרה העסקית.
- אסור "לעשות בעצמך" משימה שיש לה סוכן ייעודי.

## Open Questions

- **מתי כותבים את שאר הסוכנים הכפופים (הנה"ח, קמפיינים, אופרציה)?** הצוות השיווקי כבר מפוצל לקבצים נפרדים. השלושה האחרים עדיין רק כקטגוריות ב-`ceo.md`. צריך להחליט אם לכתוב את כולם יחד או לפי הצורך (משימה ראשונה שדורשת סוכן X → כותבים את ה-agent file שלו).
- **איך CEO מקבל תוצר מסוכן בפועל?** ב-Claude Code — דרך return value של subagent dispatch. צריך לוודא שבמשימה הראשונה זה אכן עובד טכנית כמתואר ב-PRD.
- **חיבור ל-CLAUDE.md** — מחוץ להיקף בשלב הזה לפי PRD. צריך להחליט אחרי הרצה ראשונה אם להוסיף הוראה ב-CLAUDE.md "כל משימה — קודם CEO subagent".
- **שאלות הבהרה — עד כמה אגרסיבי?** הקובץ מגדיר עיקרון + תבנית, לא enforcement. צריך לראות בהרצות אמיתיות אם זה מספיק או שצריך checklist נוקשה.
